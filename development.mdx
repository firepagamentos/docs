<Info>
  **Pré-requisito**: Instale o Node.js (versão 19 ou superior) antes de prosseguir. <br />
  Atualize para ```docs.json``` antes de prosseguir e delete o arquivo legado ```mint.json```.
</Info>

Siga estes passos para instalar e executar o Mintlify no seu sistema operacional:

**Passo 1**: Instale o Mintlify:

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**Passo 2**: Navegue para o diretório docs (onde o arquivo `docs.json` está localizado) e execute o seguinte comando:

```bash
mintlify dev
```

Uma pré-visualização local da sua documentação estará disponível em `http://localhost:3000`.

### Portas Customizadas

Por padrão, o Mintlify usa a porta 3000. Você pode customizar a porta que o Mintlify executa usando a flag `--port`. Para executar o Mintlify na porta 3333, por exemplo, use este comando:

```bash
mintlify dev --port 3333
```

Se você tentar executar o Mintlify em uma porta que já está em uso, ele usará a próxima porta disponível:

```md
Porta 3000 já está em uso. Tentando 3001 em vez disso.
```

## Versões do Mintlify

Observe que cada lançamento da CLI está associado a uma versão específica do Mintlify. Se seu site local não estiver alinhado com a versão de produção, atualize a CLI:

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## Validando Links

A CLI pode auxiliar na validação de links de referência feitos na sua documentação. Para identificar links quebrados, use o seguinte comando:

```bash
mintlify broken-links
```

## Formatação de Código

Sugerimos usar extensões no seu IDE para reconhecer e formatar MDX. Se você usa VSCode, considere a [extensão MDX para VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) para destaque de sintaxe, e o [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) para formatação de código.

## Solução de Problemas

<AccordionGroup>
  <Accordion title='Erro: Não foi possível carregar o módulo "sharp" usando o runtime darwin-arm64'>

    Isso pode ser devido a uma versão desatualizada do node. Tente o seguinte:
    1. Remova a versão atualmente instalada do mintlify: `npm remove -g mintlify`
    2. Atualize para Node v19 ou superior.
    3. Reinstale o mintlify: `npm install -g mintlify`
  </Accordion>

  <Accordion title="Problema: Encontrando um erro desconhecido">
  
    Solução: Vá para a raiz do seu dispositivo e delete a pasta \~/.mintlify. Em seguida, execute `mintlify dev` novamente.
  </Accordion>
</AccordionGroup>

Curioso sobre o que mudou na versão da CLI? [Confira o changelog da CLI.](https://www.npmjs.com/package/mintlify?activeTab=versions)
